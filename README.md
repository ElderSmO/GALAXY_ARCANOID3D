Классический Арканоид с ООП Архитектурой
О проекте
Классическая аркадная игра в жанре Arkanoid, где игрок управляет платформой, отбивает мяч и разрушает кирпичи. Проект полностью реализован с использованием принципов объектно-ориентированного программирования.

Основной геймплей:
- Управление платформой с помощью UI кнопок
- Физическое движение мяча с отскоками от стен и платформы
- Разрушение кирпичей при попадании мяча
- Система жизней (3 жизни)
-Счет за разрушенные кирпичи

Система игры:
- Генерация случайных уровней с разным расположением кирпичей
- Разные типы кирпичей (стандартные, прочные, неразрушимые)
- Определение победы при уничтожении всех кирпичей
- Game Over при потере всех жизней
- Автоматический рестарт через 3 секунды после проигрыша

Пользовательский интерфейс
- Главное меню
- Игровой экран с отображением счета и жизней
- Экран паузы
- Экран победы
Экран Game Over

Принципы ООП:
Инкапсуляция - каждый класс инкапсулирует свою логику и данные
Наследование - иерархия классов игровых сущностей
Полиморфизм - обработка разных типов объектов через общие интерфейсы
Абстракция - четкое разделение ответственности между классами

Ключевые компоненты :
GameManager - управление состоянием игры и логикой
Paddle - управляемая игроком платформа
Ball - мяч с физикой движения
Brick - система кирпичей с разными свойствами
LevelManager - генерация и управление уровнями
UIManager - управление пользовательским интерфейсом
Интерфейсы
IMovable - для всех движущихся объектов
IDamageable - для объектов, которые могут получать урон

Из минусов ООП как по мне здесь не не уровне крупных проектов, много чего можно добавить. Честно говоря реализация логики и кнопок через такие события не очень хорошая идея как по мне.
По хорошему тут бы музыку добавить, более нормальный интерфейс, эффекты и оптимизация но времени было в обрез и пришлось быстро за день проект накидать. Прошу не судить строго.
